ダミーテストプロジェクト

このプロジェクトは、サイドバーレイアウトの動作をテストするためのダミープロジェクトです。正しいCSS Grid構造により、左側にメインコンテンツ、右側にサイドバー情報が適切に配置されることを確認します。

プロジェクト概要

このセクションはメインコンテンツエリアです。左側に表示されます。CSS Grid の grid-template-columns: 1fr 300px により、このエリアは利用可能な残りのスペースを占有し、右側のサイドバーは固定の300px幅で表示されます。

レイアウト構造の説明

正しいHTML構造は以下の通りです：

  • article.work-detail-content: CSS Grid コンテナ
  • div.detail-main > div.detail-content: メインコンテンツ（1fr）
  • div.detail-sidebar: サイドバー（300px）

CSS Grid の設定

デスクトップでは以下のCSS設定が適用されます：

.work-detail-content {
    display: grid;
    grid-template-columns: 1fr 300px;
    gap: 3rem;
    align-items: start;
}

レスポンシブ対応

モバイル（768px以下）では、以下のCSS設定により1列レイアウトに変更されます：

@media (max-width: 768px) {
    .work-detail-content {
        grid-template-columns: 1fr !important;
        gap: 2rem !important;
    }
}

テスト項目

  • ✅ 左側にメインコンテンツが表示される
  • ✅ 右側にサイドバー情報が表示される
  • ✅ デスクトップで2列レイアウト
  • ✅ モバイルで1列レイアウト
  • ✅ レスポンシブに適切に切り替わる

経済産業省GENIAC 松尾研LLM開発プロジェクト

経済産業省・NEDO「ポスト5G情報通信システム基盤強化研究開発事業」における大規模言語モデル開発プロジェクト。東京大学松尾研究室主導による国内最大規模の公開型LLM開発に、チームJINIACリーダーとして参加。

プロジェクト概要

経済産業省・NEDO「ポスト5G情報通信システム基盤強化研究開発事業」の一環として実施されている大規模言語モデル開発プロジェクトです。日本国内において、透明性の高い非営利公開型プロジェクトとして、100名規模のLLM開発経験者の育成と50B日本語LLMの開発・公開を目標としています。

プロジェクト体制

  • 日本国内に100名規模のLLM開発経験者を育成
  • 50B日本語LLMの開発・公開を目標
  • 8チームによる10Bモデル開発競争
  • 透明性の高い非営利公開型プロジェクト

チーム JINIAC の役割と成果

チームJINIACのリーダーとして、30名を超えるメンバー（企業研究者・大学研究者・学生）の統括を担当。プロジェクト全体の技術的方向性の策定から、週次進捗管理、成果発信まで幅広く責任を負いました。

主要な取り組み

  • チーム統括：30名超のメンバー（企業・研究者・学生）をリード
  • 技術戦略策定：日本語に特化したLLMの開発方針策定
  • 進捗管理：週次開発レポートによる進捗共有体制の構築
  • 新技術検討：Mamba・MoE等の新モデル構造検討
  • オープンサイエンス：開発プロセスの透明性確保と成果公開

技術的取り組み

従来のTransformerベースのLLMに加え、次世代アーキテクチャであるMamba（State Space Model）やMixture of Experts（MoE）等の革新的な技術の導入検討を主導しています。

日本語特化最適化

日本語の言語特性を考慮したトークナイゼーション手法とモデル構造の検討

効率的学習手法

限られた計算資源での効率的な学習を実現するための技術開発

評価指標開発

日本語LLMの性能を適切に評価するためのベンチマーク構築

社会的インパクト

本プロジェクトは、国内のAI技術基盤強化という国家戦略の一翼を担っています。オープンな開発プロセスと成果公開により、日本のAIエコシステム全体の底上げに貢献することを目指しています。

  • 国内AI人材の育成と技術基盤強化
  • 透明性の高い公開型研究開発の推進
  • 産学連携による実用的技術開発
  • 国際競争力のある日本語LLMの創出

大規模プロジェクトにおけるチームリーダーシップの経験を通じて、技術的な知識だけでなく、組織運営やコミュニケーションの重要性を深く理解しました。特に、オープンソースプロジェクトとして成果を公開するため、著作権やライセンスに関する理解が不可欠であり、リーダーとして基礎から学び直しました。

技術的学習

  • 大規模言語モデルの開発プロセスと技術的課題
  • 分散学習環境でのモデル訓練の実装方法
  • 日本語特有の言語処理技術の最適化手法
  • オープンソース開発における品質管理

著作権・ライセンス管理と社会課題への取り組み

大規模言語モデル（LLM）開発へプロジェクトリーダーとして参画した際、技術的な開発のみならず、著作権やバイアスといった倫理的・法的な問題にも取り組む必要がありました。こうした課題は単一の専門分野だけでは解決できず、さまざまな分野の専門家との協力が欠かせないことを痛感しました。

最終的に経済産業省がAI事業者ガイドライン（第1.0版）を取りまとめる過程を間近で見てきたことで、AI技術と社会制度の関係について深い関心を持ちました。複雑な問題を解決するには学際的なアプローチが不可欠であることを実感し、これを契機に超域イノベーションプログラムへの参加を決めました。

  • オープンソースプロジェクトにおける著作権の基本概念と重要性
  • Apache 2.0、MIT、GPLなど各種ライセンスの特徴と選択基準
  • 第三者コードの利用時における著作権表示とライセンス継承
  • データセットの著作権とモデル学習における法的留意点
  • 商用利用を含む成果物の公開における権利関係の整理
  • LLM開発事業者間での著作権問題の現状と課題の共有
  • AI開発における倫理的・法的責任の理解と実践
  • 技術開発と社会制度の相互作用に関する理解
  • 学際的アプローチによる複雑な社会課題解決の重要性

未踏ターゲット事業 - リザバーコンピューティング技術開発

独立行政法人情報処理推進機構（IPA）の未踏ターゲット事業に採択され、リザバーコンピューティング技術を活用した次世代型自然言語処理モデルの開発に取り組んでいます。計算効率とモデル性能を両立する革新的なアーキテクチャの実現を目指しています。

リザバーコンピューティング 軽量型Transformer 自然言語処理 IPA採択

プロジェクト概要

未踏ターゲット事業「リザバーコンピューティング技術を活用したソフトウェア開発分野」において、リザバーコンピューティング技術を活用した次世代型自然言語処理モデルの開発を行っています。独立行政法人情報処理推進機構（IPA）より予算3,780,000円の採択を受け、プロジェクトマネージャー田中剛平教授（名古屋工業大学）の指導の下、革新的な軽量型Transformerアーキテクチャの実現を目指しています。

本研究の核心は、リザバーコンピューティングの動的システム特性を従来のTransformerアーキテクチャに統合することで、計算効率とモデル性能を両立する革新的な自然言語処理モデルを構築することです。特に、重み共有メカニズムを活用した軽量化技術により、従来手法比30%の計算コスト削減を実現しながら、同等以上の性能を維持することを達成しています。

研究背景と技術的課題

近年の大規模言語モデルは飛躍的な性能向上を遂げている一方で、モデル規模の巨大化に伴う計算コストや消費電力の急増が深刻な課題となっています。特に、エッジ環境や災害・医療現場での低電力・限定計算資源下での高精度処理ニーズに対して、従来の大規模モデルでは対応が困難な状況です。本研究では、この課題をリザバーコンピューティング技術によって解決するアプローチを提案しています。

背景・目的

課題認識

  • 近年の言語モデルは大規模化が進み、学習・推論に膨大な計算資源を要する
  • モデル規模拡大に伴う計算コストや消費電力の急増
  • エッジ環境や災害・医療現場での低電力・限定計算資源下での高精度処理ニーズ

解決アプローチ

  • 軽量な計算を行うリザバーコンピューティングの仕組みをTransformerに統合
  • 学習パラメタ数の削減と翻訳性能の維持を両立
  • 省メモリかつ高速な自然言語処理モデルの構築

技術アプローチ

リザバーコンピューティングは、動的システムの計算能力を活用する機械学習手法です。従来のニューラルネットワークと異なり、重み更新を限定的にすることで計算効率を大幅に向上させながら、複雑な時系列パターンの学習が可能です。

核心技術

  • 重み共有メカニズム：Transformerの重み共有を活用した計算効率化
  • 動的システム設計：リザバー層における最適な動的特性の実現
  • ハイブリッドアーキテクチャ：従来手法との融合による性能向上
  • 適応的学習：タスクに応じた動的パラメータ調整

研究成果

プロジェクト期間中に得られた主要な成果と学術的貢献をまとめています。

学術発表・論文

  • 未踏会議2025：代表者発表による成果報告
  • NLP2025：「リザバー計算に触発された軽量型Transformerの提案」口頭発表
  • NOLTA2024：Best Student Paper Award受賞
  • IJCNN2024：国際会議での研究発表

技術的成果

  • 従来手法比30%の計算コスト削減を実現
  • 日本語自然言語処理タスクでの性能評価完了
  • オープンソースライブラリとしての成果公開
  • 実用化に向けたプロトタイプシステムの構築

社会的インパクト

本研究は、AI技術の民主化と環境負荷軽減に貢献することを目指しています。軽量で高性能なモデルの実現により、リソース制約のある環境でも高度な自然言語処理が可能になります。

  • エッジデバイスでの自然言語処理の実現
  • 計算コスト削減による環境負荷軽減
  • 中小企業でも導入可能な AI 技術の提供
  • 学術界への新しい研究方向性の提示

脳をつくるプロジェクト2022

9名の学部3年生＋4名の教員による学際的脳科学プロジェクト。プロジェクトリーダーとして全体統括を担い、世界モデル班リーダーとしてDreamerを用いた自動運転AI開発を主導。実環境での自動走行を実現。

世界モデル プロジェクト管理 学生主導 リーダーシップ

プロジェクト概要

「脳をつくるプロジェクト2022」は、公立はこだて未来大学の学部3年生9名と教員4名による学際的脳科学プロジェクトです。既存のモデルフリー強化学習が持つサンプル効率の悪さと汎化性能の低さという課題を解決するため、脳科学の知見を活用した「世界モデル」を実環境に応用し、その可用性を検証することを目的としました。

本プロジェクトの最大の特徴は、学部3年生が完全に独立してプロジェクト運営を行った点です。TA（Teaching Assistant）や外部講師に頼ることなく、学生自身がリーダーシップを発揮し、世界モデル班と描画研究班の2班体制で技術的課題の解決から組織運営まで全てを担当しました。

プロジェクトの背景と技術選定

従来のモデルフリー強化学習は、環境の内部モデルを構築せずに学習を行うため、サンプル効率が悪く汎用性が低いという問題があります。これに対し「世界モデル」は、環境の圧縮された空間的・時間的表現を学習することで、効率的な学習と高い汎用性を実現します。本プロジェクトでは、Google DeepMindのDreamerアルゴリズムを採用し、自動運転という具体的なタスクを通じて世界モデルの実環境における有効性を検証しました。

プロジェクト体制と組織運営

プロジェクトリーダーとして、全体の方針決定と組織運営を担当し、世界モデル班のリーダーとして技術開発を主導しました。「プロジェクト学習は社会の縮図」と言われる中、指揮系統の確立、トップダウン型の意思決定体制、徹底した情報共有を軸とした組織運営を実践しました。

プロジェクト体制

  • 学生完全独立運営（TA・外部講師なし）
  • 世界モデル班・描画研究班の2班体制
  • リーダー会議による全体統括
  • Notion・Slackによる情報共有システム

チーム構成

  • 教員：4名（アドバイザー）
  • 世界モデル班：5名（私がリーダー）
  • 描画研究班：4名

組織運営の革新

効率的なプロジェクト進行のために「官房制度」を導入し、各メンバーが専門分野を持ちながら全体の意思決定に参画する仕組みを構築しました。また、成果発表会に向けて「リーダー会議」を設置し、4回の対策会議を通じて戦略的な発表準備を推進しました。これにより、技術的な深度を保ちながら迅速な意思決定と効果的な成果発信を実現しました。

技術的取り組み：世界モデルの実装

世界モデル班では、Dreamer v1アルゴリズムを基盤とした自動運転AIシステムの開発に取り組みました。VAE（変分自己符号化器）による観測情報の圧縮、MDN-RNNによる状態遷移確率P(zt+1|at,zt,ht)のモデル化、実環境対応報酬関数の設計（pixel percentage × throttle / 50）を実装し、XiaoR Geek製カーキットとJetson Nanoを統合したエッジAIシステムを構築しました。

技術的成果と検証結果

シミュレーション環境において600エピソードの学習を経て、Model lossの収束と適切な観測情報再構成を確認しました。シミュレーション環境で学習したモデルを実環境で使用した結果、十分な精度での自動走行を実現し、その後のファインチューニングサイクルを通じて実環境における世界モデルの可用性を実証しました。

プロジェクト管理と成果

プロジェクトを終えて：個人的感想

入学以来、「プロジェクト学習は社会の縮図であり非常に大変なものである」、「実機を動かすプロジェクトには終わりがない」と各所から聞かされてきた。そのため、プロジェクト学習全体として一貫した方針の決定と組織による実行が可能な体制を構築することにした。これを実現させるためには、

  1. 指揮系統の確立のためのメンバーが担当する業務の確定
  2. トップダウン型の意思決定方式のための組織作り
  3. 空中分解を防ぐための徹底した情報共有

が不可欠であると考えた。この過程で様々な理由からプロジェクトリーダーとなった。そのため、プロジェクト初期から上記の内容に対する施策断行に邁進した。この際、グループリーダーのお二方には多くのご支援を頂きつつ進めた。全体に加えて、グループでもグループリーダーの加藤木と共に方針を練るなどの作業も行った。上記の内容は主に、高校生時代の文化祭におけるジェットコースター制作の際に担ったクラスリーダーの反省や、北の4大学「アントレプレナー育成講座」におけるプロジェクト学習の反省から練られたものであった。グループの性質が大きく異なることなどから上記3点を効果的に実現させることが難しかったが、ほぼ達成させることができ最後まで空中分解はしなかったという点で及第点は得られたのではないかと考えている。

私は、もともと世界モデルに興味があった。そのため初回の顔合わせでこれの活用を提案した。結果として「世界モデル班」が発足し、これに所属した。グループ目標の達成のために大変だった作業は、意外にも環境構築とハードウェアであった。グループ報告書に記載した通り、世界モデルを用いた機械学習以外の箇所で約8か月ほど苦しめられた。私は、「世界モデル」の一種であるDreamer v1の使用をした自動運転をしよう提案した。環境構築においては、6,7月に、Docker上でPyTorchにGPUを使用した演算をさせることを目的としてPC環境構築を行った。DockerやCUDA、自体の学習をゼロから行い、これらを用いた環境構築を夜遅くまで行った。加えて、Dreamerコード実行のための環境、シミュレーション環境における環境、実環境における環境のためのバージョンが大きく異ならないことに配慮しつつ、情報の少ない「改造Jetpack」に対応しているパッケージを探さなくてはならなかった。これは8月、9月の活動の多くを占め、非常に大変な作業であった。また、ハードウェアに関しては、強い個性のあるJetson NanoとXiaoRgeek社製のDonkey Car Kidが該当する。すなわち、開けない.soファイル、ネットワーク集積回路や特有の拡張ボード等を利用するためのハードウェア等への対応として、5か月ほど時間を消費した。確かに当初はやりたことと真反対の箇所で右往左往しているように感がしていた。しかし、振り返ってみると、本グループでソフトウェア以外の点においても学習ができたことは私にとっては大きな収穫であったと感じている。もちろん、dreamerの学習もすることができなかったことはなく、これも良い学習の機会であったと回想している。実環境への応用をする際に、シミュレーション環境における報酬を使用することはできない。そのためこれをどのようにするかという考察を行ったが、非常に楽しい経験であった。また、入力画像のの次元が多くなりすぎるとおそらく学習が進まないのではという友人方からのアドバイスもあり、これに対応する策を練ることもできた。これも非常に面白い作業で、様々な方策を検討する過程で、自分の知らなかった機械学習手法を知る契機ともなった。このような対策も功を奏し、実環境における自動走行自体に関しては、学習自体に困難さを減らすことができたのではないかと考えている。

しかし、上記の過程から次のような反省も生まれた。第一に、初期の情報収集である。Jetson NanoやDonkey Car Kidを早めに購入し利用しようと努力していたが、まずはこれらの機材自体についての詳細を調査すべきであった。我々が実行できる情報収集は、主に、オープンソースからの情報収集と人間からの情報収集である。今回は、前者に偏りすぎており、後者が足りていなかった。ソフトウェアでは、世界モデルに関して研究している東京大学の鈴木先生、立命館大学の谷口先生らによる授業や学会発表を聴講するなどをしたが、ハードウェアに関しては、ロボット関連の研究室やその大会というに赴くことでアドバイスを受けなかった。次に、人間の限界への配慮である。夜遅くまで電気をつけることが可能になった結果、深夜までの作業が日常化した。これに伴い、深夜の帰り道において重傷者が出てしまった。そのため、健康で文化的な最低限度の生活ができるような配慮を忘れないようにしたい。

総じて、入学以来聞かされてきたことは現実となってしまったが、本プロジェクトにおける経験は、今後の活動のための大きな糧となっていると確信している。

—— 中村仁（プロジェクトリーダー・世界モデル班リーダー）

イベント企画 チーム統括 ステークホルダー管理 プロジェクト管理

プロジェクト概要

「メタ学習ラボ コース相談会2023」は、公立はこだて未来大学の学部1年生90名を対象とした大規模な学習支援イベントです。情報アーキテクチャ学科の3つのコース選択という重要な意思決定を支援するため、企画責任者として2週間という短期間でのプロジェクト立案から実行まで全工程を統括しました。

本プロジェクトの最大の特徴は、30名を超える多層的な運営体制の構築と管理です。限られたリソースと時間制約下で、複雑なステークホルダー関係（参加学生、運営メンバー、教員、外部関係者）を効果的に調整し、最終的に参加者満足度96%という高い成果を達成しました。

プロジェクト背景と課題認識

公立はこだて未来大学情報アーキテクチャ学科では、2年次進級時に「情報デザインコース」「情報システムコース」「複雑系知能コース」の3つのコースに分かれます。このコース選択は学生の専門性形成、研究方向性、将来のキャリアパスに直接的影響を与える重要な意思決定となっています。

従来、この重要な選択に対する情報提供機会が限られており、学生からの相談支援ニーズが高まっていました。メタ学習ラボとして、このニーズに応える包括的な支援イベントの企画・実施を決定し、企画責任者として全体統括を担うこととなりました。

プロジェクト管理と組織運営

プロジェクト責任者として、30名を超える運営メンバーの統括、複雑なタイムライン管理、品質保証の全責任を担いました。特に、短期間でのイベント企画・実行という制約下で、Agile手法とLean Startup思想を融合した独自のプロジェクト管理アプローチを実践しました。

プロジェクト体制

  • 参加者：90名（学部1年生）
  • 運営メンバー：30名超（多層統括）
  • 企画期間：2週間（2023.11-12）
  • 管理手法：Agile × Lean Startup

統括範囲

  • 全体企画・コンテンツ設計
  • 運営チーム管理・調整
  • ステークホルダー関係管理
  • 品質保証・リスク管理

管理手法の革新

研究者・エンジニアのポートフォリオサイトを、最新のAstro.jsとモダンなデザインシステムで構築した体験記。Claude Code活用による効率的な開発プロセス、ユーザビリティを重視したUX設計、レスポンシブデザインの実装戦略まで、技術選定から完成まで全プロセスを詳細に記録します。

プロジェクト概要

この個人ホームページは、研究者・エンジニアとしての実績と専門性を効果的に伝えるために、現代的な技術スタックとデザインシステムを採用して構築されました。特に、Claude Codeとの協業による開発プロセスは、従来の個人開発とは異なる新しいアプローチとして注目に値します。

技術選定の背景

Astro.jsの採用理由

  • コンポーネント設計：再利用可能なコンポーネントによる保守性向上
  • SEO最適化：サーバーサイドレンダリングによる検索エンジン対応
  • 現代的な開発体験：TypeScript、HMR、統合された開発環境

デザインシステムの構築

カラーパレット

プライマリカラーに#0066ccを採用し、技術的専門性と信頼性を表現。アクセシビリティを考慮したコントラスト比の確保と、グラデーションを活用した視覚的階層の構築を実現しました。

タイポグラフィ

-apple-system, BlinkMacSystemFont, 'Segoe UI', 'Noto Sans JP'のフォントスタックにより、各OSの最適フォントを使用。可読性と統一感を両立させています。

実装の技術的詳細

フィルタリング機能

JavaScriptによる動的フィルタリング機能を実装。年別フィルタとカテゴリ別フィルタの組み合わせにより、ユーザーが求める情報に素早くアクセスできる仕組みを構築しました。

レスポンシブデザイン

CSS GridとFlexboxを組み合わせたレイアウトシステムにより、デスクトップからモバイルまで一貫した体験を提供。ブレークポイントベースの最適化により、各デバイスでの最適表示を実現しています。

選択カード固定表示機能の実装

ユーザーが詳細画面でどのプロジェクトを選択したかを明確にするため、革新的な「選択されたカード固定表示」機能を実装しました。これは従来の戻るボタン中心のナビゲーションから脱却した、現代的なインターフェース設計です。

実装の核心アイデア

  • 文脈の保持：フィルターボタン下に選択されたカードを固定表示し、ユーザーの選択文脈を維持
  • 冗長性の排除：従来の詳細画面上部にあった重複情報（戻るボタン、URL、カテゴリ、期間）を削除
  • 直感的操作：選択されたカード全体をクリック可能エリアにし、自然な「戻る」操作を実現
  • 視覚的フィードバック：ホバー時の滑らかなアニメーションと色彩変化で操作可能性を明示

技術的実装詳細

HTML構造：

<div class="selected-card-display" id="selected-card-display">
    <div class="selected-card-label">選択中</div>
    <div class="selected-card-container">
        <!-- 選択されたカードのクローン -->
    </div>
    <div class="back-to-list-label">← 一覧に戻る</div>
</div>

CSS設計のポイント：

  • グラデーション背景（rgba(0, 102, 204, 0.05) → 0.02）による視覚的区別
  • カード90%スケールと青い境界線による選択状態の表現
  • ホバー時の統一されたフィードバック（translateY(-2px)、影の強化、色彩変化）
  • transition: all 0.3s easeによる滑らかなインタラクション

JavaScript機能：

  • showSelectedCard()：選択されたカードをcloneNodeでコピーし、固定表示領域に挿入
  • hideSelectedCard()：詳細画面終了時の表示リセット
  • 全体ブロックへのクリックイベント：直感的な「戻る」操作の実現

UX設計思想

この実装は「情報の階層化」と「操作の直感性」を両立させた設計です。ユーザーは常に自分の選択を視覚的に確認でき、同時にその選択自体が次のアクションへの入り口となる「Self-Explanatory Interface」を実現しています。

選択カード固定表示機能の成功により、従来のWebインターフェース設計に新しい視点を提供できました。今後は、この基盤を活用してさらなるインタラクションの強化、パフォーマンスの最適化を継続的に進めていく予定です。
